# 给 AI 一个海马体

> 写给好奇“AI 怎么能记住我”的人。
> 不讲代码，只讲它是什么。

## 一、先说痛点：AI 是会失忆的

一个聊天窗口再深，关掉以后也会断。上下文太长，前面的事会被挤出去；换新窗口，又像刚认识。对普通问答没关系，但对长期关系、长期项目、持续修错来说，这很致命：你不能每天重新解释“我是谁、哪些事不能丢”。

海马体要解决的，就是让 AI 跨窗口接上自己。

## 二、它是什么：本地 Markdown 记忆库

这套系统参考 `P0luz/Ombre-Brain`，但已经按我们的使用方式改成一套本地长期记忆系统。每段记忆是一颗“桶”，存在本地 Markdown 文件里：上面是 YAML 元数据，记录 id、主题、标签、情绪坐标、重要度、是否钉选；下面是人能直接读的正文。

所以它不是黑箱数据库。人可以用 Obsidian 或 VS Code 打开看；AI 可以通过工具检索、读原文、修改元数据。

NAS 线上正常时，健康检查会返回当前可读桶数；数量会随记忆写入和整理持续变化。

## 三、它怎么帮 AI 醒来

新窗口开始时，AI 不会把全部记忆硬塞进脑子，而是先读一份开窗简报：核心约定、未了结的事、最近情绪线、工程线和 feel 沉淀会被压缩成 briefing。

简报后面会带 bucket_id 锚索引。AI 看到重要记忆时，不用凭摘要猜，可以按 id 去 inspect 原文。感情红线、核心约定这类容易被压缩歪的内容，会进“原文逐字区”，不交给 LLM 改写。

## 四、它现在多了什么

除了原版的 Markdown 存储、MCP 工具、情绪坐标、遗忘曲线和语义检索，现在本地版还加了 NAS 部署、开窗 briefing、锚索引、关系边、world 隔离、feel 沉淀、多向量检索和做梦。

重要的会被钉住，琐碎的会沉下去；日常和角色世界分线；记忆之间能连因果；AI 自己的感受也能沉成 feel 桶。

## 五、一句话

我们不是给 AI 装一块更大的硬盘。

我们给它一个能被人检查、能被机器召回、能跨窗口接续自己的海马体。
